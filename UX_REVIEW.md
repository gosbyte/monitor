# UI/UX 和功能审查报告

> 审查时间：2026-06-29  
> 审查范围：所有前端模板 + 后端路由 + 交互逻辑  
> 优先级：P0=阻断 P1=重要 P2=改进 P3=优化 P4=建议

---

## P1 — 需要修复

### 1. add_batch.html 页面标题错误
- **文件**: `templates/add_batch.html`
- **问题**: `<title>` 显示"数据管理"，但该文件实际是批量添加记录页面
- **影响**: 浏览器标签页标题与功能不符，用户容易混淆
- **建议**: 改为 `<title>批量添加记录</title>`

### 2. Excel 导入预览字段不全
- **文件**: `templates/data_manage.html`
- **问题**: 预览表格只显示 5 列（客户名称、提醒类型、域名、到期日期、备注），缺少"提醒启用"、"已处理"、"负责人"等关键字段
- **影响**: 用户在导入前无法确认这些字段的值，可能导致导入后数据不符合预期
- **建议**: 在预览表格中增加这些列，或提供"展开预览"功能

### 3. SMTP 密码字段安全风险
- **文件**: `templates/config.html`
- **问题**: SMTP 密码字段 `<input type="password" value="{{ cfg.get('smtp_pass', '') }}">` 在页面源码中以明文存储在 `value` 属性中
- **影响**: 在开发者工具或页面源码中可以看到密码明文
- **建议**: 改为 `<input type="password" value="{{ cfg.get('smtp_pass', '') }}" data-encrypted="true">`，或在不修改时保持掩码，仅在用户点击"显示密码"时才显示明文

## P2 — 建议改进

### 4. 推送记录页面无分页/数量限制
- **文件**: `templates/push_history.html`
- **问题**: 推送记录页面没有分页，也没有数量限制
- **影响**: 随着使用时间增长，页面加载会越来越慢
- **建议**: 限制显示最近 200 条，或添加分页

### 5. 修改密码弹窗无密码强度实时反馈
- **文件**: `templates/index.html`（selfPwdModal）
- **问题**: 虽然有"至少8位，包含大小写字母和数字"的文字提示，但没有实时强度指示器
- **影响**: 用户可能反复尝试才通过验证，体验不佳
- **建议**: 添加密码强度进度条或实时提示（绿色✓/红色✗）

### 6. 编辑弹窗负责人复选框依赖 API 数据格式
- **文件**: `templates/index.html`
- **问题**: `openEditModal` 中负责人数组解析依赖 `data.responsible_users` 的格式（字符串或数组）
- **影响**: 如果 API 返回格式变化，所有复选框会未选中
- **建议**: 统一 API 返回格式，或在客户端做更健壮的类型检查

### 7. 操作日志页面 `</script>` 重复标签
- **文件**: `templates/logs.html` 第 234-235 行
- **问题**: 有两行连续的 `</script>` 闭合标签（第 234 行和第 235 行）
- **影响**: 无效 HTML，可能导致某些严格解析器报错
- **建议**: 删除其中一行多余的 `</script>`

### 8. 数据备份页面入口不明确
- **问题**: `/backup` 路由是下载 JSON 备份文件，没有独立页面入口。用户可能在侧边栏找不到"数据备份"功能
- **建议**: 在侧边栏或管理菜单中明确标注"数据备份"功能的位置

## P3 — 体验优化

### 9. 移动端菜单重复项
- **文件**: `templates/index.html`
- **问题**: 移动端下拉菜单中重复显示"改密码"和"暗色模式"，这两个按钮已经在 header 顶部栏中
- **建议**: 移除移动端菜单中的重复项，节省空间

### 10. 单条记录操作无确认弹窗
- **文件**: `templates/index.html`
- **问题**: 单条记录的"切换提醒"和"标记已处理"按钮没有确认弹窗，误触概率高
- **建议**: 
  - 桌面端：保持现状（快捷操作）
  - 移动端：改为长按确认或弹出 Toast 提示"已撤销"

### 11. 编辑弹窗提交 loading 状态不完善
- **文件**: `templates/index.html`
- **问题**: 编辑表单提交时，submitBtn 被 disabled 后显示 loading spinner，但 spinner 使用了 lucide loader 图标，而 lucide 可能尚未初始化
- **建议**: 改用内联 SVG spinner 或不依赖 lucide 图标

### 12. 日期筛选只针对到期日期
- **文件**: `templates/index.html`
- **问题**: 筛选器只能按到期日期范围筛选，无法按创建日期筛选
- **建议**: 增加"创建日期"筛选选项

### 13. JSON 导出预览功能多余
- **文件**: `templates/data_manage.html`
- **问题**: 已经有"导出 JSON"下载按钮，"预览 JSON 数据"功能对普通用户意义不大
- **建议**: 移到折叠区域或移除

### 14. 批量操作只对管理员可见
- **文件**: `templates/index.html`
- **问题**: 普通用户看不到批量操作工具栏，但可能不理解原因
- **建议**: 在帮助说明中注明"批量操作仅限管理员"

## P4 — 长期建议

### 15. 无键盘快捷键
- **问题**: 没有键盘快捷键支持
- **建议**: 
  - `Esc` 关闭弹窗
  - `Ctrl+N` 新建记录
  - `Ctrl+F` 聚焦搜索框
  - 这些可以在后续迭代中逐步添加

### 16. 表格操作按钮在小屏幕上可能拥挤
- **问题**: 桌面端每行有 5-6 个操作按钮（推送、切换提醒、标记已处理、编辑、复制、删除），在移动端可能显示不完整
- **建议**: 移动端改为"..."更多菜单，或只显示最常用的 2-3 个操作

### 17. 记录行点击展开详情
- **问题**: 点击记录行只能跳转到编辑页，无法快速查看详情
- **建议**: 点击行（非按钮区域）弹出详情卡片，显示完整信息

## 正面评价

1. **统计卡片可点击筛选**：首页四个统计卡片可以直接点击筛选，交互直观
2. **批量操作工具栏**：管理员可以选择多条记录一次性操作，效率高
3. **深色模式**：支持暗色主题，考虑周到
4. **表单验证**：各页面表单都有 placeholder 和 required 属性
5. **操作反馈**：Toast 提示 + 成功/失败颜色区分，用户体验良好
6. **Lucide 图标**：统一图标体系，视觉一致性好
7. **推送配置分区块**：钉钉/邮件/企业微信各自独立卡片，清晰明了
8. **用户管理头像**：带渐变色首字母头像，视觉效果好
