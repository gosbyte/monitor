#### P1 — 重要问题

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| 4 | **CSP 被注释掉** | app.py:144-152 | 安全加固被回退，手机浏览器兼容性问题未根本解决 |

## 三、代码质量分析

### 3.1 优点

1. **登录安全做得很好** — 限流 + 锁定 + 枚举防护 + session 固定防护
2. **验证码实现合理** — Pillow 生成 + 干扰线 + 4位混合字符
3. **daemon.py 提醒逻辑完善** — 支持多频道推送 + 负责人定向邮件 + 状态去重
4. **文件锁保护并发** — fcntl.flock 防止 JSON 文件损坏
5. **Prometheus 指标集成** — 方便监控告警
6. **备份/恢复/导入导出** — 数据可移植性好

### 3.2 问题

1. **db.py 写了 415 行 SQLite 代码但从未被调用** — 这是最大的架构浪费
2. **data.py (352行) 和 db.py (415行) 功能重复** — 两套 CRUD 并存
3. **app.py 1324 行过于臃肿** — 单文件承载了路由、业务逻辑、API
4. **init_data.py 和 data.py 双重初始化** — 用户初始化逻辑分散
